Lo Complicado de las SPAs

  • Arquitectura clásica: React, Vue, Angular.
  • El Backend se convierte en solo una API (JSON).
  • El Frontend consume esa API y arma la interfaz.
  • El Problema: Demasiada complejidad. Dos equipos, gestión de APIs, sincronización, sobre-ingeniería.
  • Es una complejidad innecesaria, como usar una máquina pesada para un trabajo simple.

La Alianza Ideal

Django <3 HTMX
Rescatando lo mejor de antes, pero con la agilidad de hoy.

¿Cómo Funciona Esta Propuesta?

1. El Usuario Toca

Un usuario hace clic en un botón con un atributo HTMX simple (ej: `hx-post`).

2. Pide al Servidor

HTMX manda una petición a Django (al backend).

3. Django Devuelve HTML

¡Sin JSON! Django renderiza un *pedacito* de HTML y lo manda de vuelta.

4. HTMX Lo Pone

HTMX agarra el HTML y lo 'inyecta' donde va en la página. Sin recargar, al instante.

Ejemplo de Código: El HTML

El "JavaScript" que va (en serio):

<!-- Sí, en serio. Esto es todo el código. -->
<script src="https://unpkg.com/htmx.org@1.9.10"></script>

Sin builds, sin webpack, sin gigas de `node_modules` (¡qué alivio!).

"Un Momento... ¿Y el JavaScript?"

HTMX: "Menos JavaScript, Más HTML"

  • Manejo de Eventos: ¿Querés que un 'submit' salte con un 'change' o 'keyup'?
    hx-trigger="change" o hx-trigger="keyup delay:500ms"
  • Polling: ¿Actualizar un div cada 5 segundos? (¡Genial para notificaciones en vivo!)
    hx-trigger="every 5s"
  • Validación de Formularios: Validar campos *antes* de mandar, sin recargar y solo reemplazando el error.
    hx-swap="outerHTML"
  • Tenés la ventaja: El 90% de lo que hacías con JS para "manipular el DOM" ahora está en atributos HTML declarativos, ¡súper simple!

La Ventaja: Menos Complicaciones

1
Base de Código Única

¡Adiós, API Complicada!

Eliminás el dolor de cabeza de tener una API solo para el frontend. La interfaz y el servidor se comunican en HTML. Esto hace el desarrollo mucho más rápido y el mantenimiento mucho más sencillo.

La Ventaja: Productividad Máxima

  • Trabajás en un solo lugar: Te concentrás en tu proyecto Django. Se terminan los cambios de contexto entre repositorios, lenguajes y compilaciones.
  • Aprendés rápido: Si ya conocés Django y sus templates, ya tenés el 80% hecho. HTMX lo incorporás en una tarde.
  • Agilidad: Sacás productos y características al mercado muchísimo más rápido. Es ideal para MVPs, startups y equipos pequeños.

La Ventaja: Rendimiento y SEO

  • Muy ligero: Las aplicaciones tienen tiempos de carga inicial mucho menores. El navegador recibe HTML listo, no un JS que *recién después* renderiza.
  • Amigo del SEO: El contenido ya viene en el HTML, el cliente no lo tiene que generar. ¡Esto es *fantástico* para que te encuentren!
  • Fácil de indexar: Los robots de Google entienden todo al instante, sin tener que ejecutar JavaScript complicado.
